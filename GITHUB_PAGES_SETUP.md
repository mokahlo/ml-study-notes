# GitHub Pages Setup for CEE 501 Study App

## Step-by-Step Setup (5 minutes)

### Step 1: Create GitHub Account
- Go to https://github.com
- Click "Sign up"
- Create account with your email
- Verify email

### Step 2: Create Private Repository
1. Click the **+** icon in top right ‚Üí **New repository**
2. Fill in:
   - **Repository name:** `cee501-study` (or any name)
   - **Description:** CEE 501 Midterm 2 Study Companion
   - **Select:** Private ‚≠ê (important - only you can see it)
   - Leave other options default
3. Click **Create repository**

### Step 3: Upload Your HTML File
1. You should see an empty repo page
2. Click **Add file** ‚Üí **Upload files**
3. Drag and drop `CEE501_Study_App.html` into the browser window
4. Or click "choose your files" and select it
5. In "Commit message" write: `Add study app`
6. Click **Commit changes**

### Step 4: Enable GitHub Pages
1. Click **Settings** (tab at top of repo)
2. Scroll down to **Pages** section (left sidebar)
3. Under **Source**, select **Deploy from a branch**
4. **Branch:** select `main`
5. **Folder:** select `/ (root)`
6. Click **Save**
7. Wait ~1 minute for deployment

### Step 5: Get Your URL
1. Go back to **Settings** ‚Üí **Pages**
2. You'll see a message: "Your site is live at: `https://yourusername.github.io/cee501-study/`"
3. The full URL to your app: `https://yourusername.github.io/cee501-study/CEE501_Study_App.html`

### Step 6: Access on Phone
- Copy that URL
- Open on your phone browser
- Bookmark it!
- Your progress auto-saves locally on each device

6. Refresh browser (1 min for GitHub to deploy)

## Troubleshooting

**"Site not found"?**
- Wait 5 minutes for GitHub to deploy
- Check URL spelling (yourusername, not "username")
- Refresh page (Ctrl+Shift+R on computer, hard refresh on phone)

**Progress not syncing between devices?**
- This is expected! Each device has separate storage
- You can manually transfer progress via export (would need to add feature)

**Want to share with study partner?**
- Change repo to "Public"
- Share URL with them
- OR add them as collaborator (Settings ‚Üí Collaborators)
